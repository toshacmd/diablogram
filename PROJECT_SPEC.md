# Нейрокомментинг для Telegram — спецификация проекта

Личный (single-tenant) сервис для автоматического оживления обсуждений на собственных Telegram-каналах через ИИ-комментарии от пула управляемых аккаунтов.

## Контекст

- Владелец имеет **50+ собственных Telegram-каналов**.
- Цель — не реклама и не перекрёстное продвижение между каналами, а **создание видимости живого обсуждения** под постами.
- Инструмент личный, продавать доступ другим не планируется (мультитенантность не нужна).
- Все каналы — на русском языке.

## Основной цикл работы

1. На канале выходит новый пост → сервис детектирует событие.
2. Если пост **без текста** (только фото/видео без подписи) — комментирование пропускается. Анализ медиа (vision) не делается.
3. Из аккаунтов, закреплённых за этим каналом, выбирается количество комментаторов по **настраиваемому диапазону** (например, 1–3). Нижняя граница диапазона — **минимум 1**: пост без единого комментария не допускается.
4. Каждый выбранный аккаунт публикует комментарий с индивидуальной **задержкой**, заданной пользователем как диапазон "от–до" (не точное фиксированное значение, не хардкод системы) — чтобы комментарии не выходили одновременно.
5. Текст комментария = **ИИ-генерация по контексту поста** (с учётом тона/личности конкретного аккаунта) + **статичная фирменная подпись** этого аккаунта.
6. Перед публикацией — автоматический **контент-фильтр** (стоп-слова/темы, проверка тональности), с возможностью **отключения**.
7. Публикация — **полностью автоматическая**, без очереди ручного подтверждения/модерации.
8. Ответы реальных читателей на комментарии бота (треды-диалоги) — **вне scope первой версии**, не отслеживаются и не обрабатываются.

## Управление аккаунтами

- **Подключение аккаунта**: импорт готовой session-строки (Telethon/MTProto), без интерактивного логина (номер+код) через панель.
- **Прокси**: индивидуальный прокси настраивается на каждый аккаунт отдельно.
- **Закрепление за каналами**: гибкое — от одного канала до всех сразу, произвольные подмножества.
- **Тон/личность**: переключатель на каждом аккаунте — набор готовых пресетов + возможность создать свой кастомный промпт-персону.
- **Подпись**: уникальная фирменная подпись на каждый аккаунт, статичный текст добавляется к каждому ИИ-комментарию этого аккаунта.
  - Редактирование **точечное** (один аккаунт) и **массовое** (сразу на всех) — обе функции обязательны.
- **Суточный лимит**: настраиваемый потолок количества комментариев в день на аккаунт — защита от совокупной перегрузки аккаунта независимо от числа закреплённых каналов.
- **Статусы аккаунта**: автоматические — активен / временный лимит (FloodWait) / бан.
  - Без попыток автоматической "реанимации" забаненных аккаунтов — только фиксация статуса.
  - Уведомление о смене статуса — в Telegram (служебным ботом владельцу) + отображение статуса в веб-панели.

## ИИ-генерация

- Провайдер — **платный API** (Claude или OpenAI; конкретная модель — решение этапа реализации, не блокирует архитектуру).
- Self-hosted/открытые модели рассмотрены и отклонены на данном этапе (короткие комментарии дешевле через API, чем содержание своего GPU-сервера).
- Язык генерации — русский (захардкожен, мультиязычность не нужна).
- Учитывается тон/персона аккаунта при генерации (не единый нейтральный стиль на всех).

## Интерфейс

- **Веб-панель** (не Telegram-бот, не конфиг-файлы) — из-за объёма табличных настроек (каналы, аккаунты, подписи, персоны, диапазоны) и необходимости массовых операций.
- Разделы: каналы, аккаунты (с прокси, session, лимитами, статусами), персоны/пресеты тона, подписи (точечное/массовое редактирование), настройки диапазонов (количество комментаторов, задержка).
- **Журнал (лог)** всех опубликованных комментариев: кто (аккаунт), когда, где (канал/пост), что написал.
- Минимальная статистика на основе журнала (без сложной аналитики на старте).

## Технический фундамент

- **Telethon** (Python, MTProto) — библиотека для мультиаккаунтной работы от лица пользовательских Telegram-аккаунтов (не Bot API) и отслеживания событий новых постов.
- Готового open-source решения под весь набор требований не найдено. Ближайшие аналоги (закрывают только базовый цикл "пост → комментарий" одним аккаунтом, без мультиаккаунтности/панели/персон):
  - [ArnoGevorkyan/Telegram-AI-Commenter](https://github.com/ArnoGevorkyan/Telegram-AI-Commenter) (Node.js, GPT-4o-mini)
  - [SastaDev/Auto-Channel-Comment-Telegram-User-Bot](https://github.com/SastaDev/Auto-Channel-Comment-Telegram-User-Bot) (Python, Telethon)

## Явно вне scope (первая версия)

- Мультитенантность / продажа доступа другим пользователям
- Анализ медиа (vision) для постов без текста
- Обработка ответов/тредов на комментарии бота
- Self-hosted ИИ-модели
- Ручная модерация комментариев перед публикацией (даже опционально)
