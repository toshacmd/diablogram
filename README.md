# Diablogram AI — нейрокомментинг для Telegram

Личный сервис для оживления обсуждений на собственных Telegram-каналах: при
выходе нового поста ИИ генерирует контекстный комментарий от лица одного из
управляемых аккаунтов и добавляет к нему фирменную подпись этого аккаунта.

Полная спецификация — [PROJECT_SPEC.md](PROJECT_SPEC.md).

## Компоненты

- **web** — панель управления (FastAPI): каналы, аккаунты, персоны, подписи, настройки, журнал.
- **worker** — фоновый процесс (Telethon): держит аккаунты на связи, слушает новые посты, генерирует и публикует комментарии по расписанию.
- **db** — PostgreSQL.

## Требования

- Docker + Docker Compose (для продакшена на VPS).
- Python 3.12 (для локальной разработки без Docker).
- API ID/Hash с [my.telegram.org](https://my.telegram.org) — один на всё приложение, используется всеми аккаунтами.
- Бот-токен от [@BotFather](https://t.me/BotFather) — для уведомлений владельцу (отдельный от аккаунтов-комментаторов).
- Платный API-ключ ИИ-провайдера — Anthropic или OpenAI.

## Настройка

1. Скопируйте `.env.example` в `.env` и заполните:
   - `DATABASE_URL` (для docker-compose можно оставить значение по умолчанию)
   - `TELEGRAM_API_ID`, `TELEGRAM_API_HASH`
   - `NOTIFIER_BOT_TOKEN`, `NOTIFIER_OWNER_CHAT_ID` (свой Telegram user id, узнать можно через [@userinfobot](https://t.me/userinfobot))
   - `AI_PROVIDER` (`anthropic` или `openai`) и соответствующий `*_API_KEY`
   - `SESSION_ENCRYPTION_KEY` — сгенерировать командой:
     ```
     python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
     ```

2. Запуск:
   ```
   docker compose up -d --build
   ```
   Сервис `migrate` применит миграции БД перед стартом `web`/`worker`.

3. Откройте панель: `http://<сервер>:8000`.

## Добавление аккаунта

Каждому аккаунту-комментатору нужна session-строка (Telethon StringSession).
Получить её:

```
python scripts/generate_session.py
```

Скрипт запросит номер телефона / код / пароль 2FA (стандартный вход в
Telegram) и напечатает session-строку — вставьте её в панели при добавлении
аккаунта, вместе с прокси (если используется).

## Типичный порядок настройки

1. **Аккаунты** — добавьте аккаунты (session-строка, прокси, суточный лимит, подпись, персона).
2. **Каналы** — добавьте канал по username, указав любой уже добавленный аккаунт для получения данных из Telegram.
3. На странице аккаунта — закрепите за ним нужные каналы.
4. **Персоны** — при необходимости добавьте свои тона поверх готовых пресетов.
5. **Настройки** — диапазон количества комментаторов на пост, диапазон задержки, контент-фильтр и стоп-слова.
6. Проверьте **Журнал** — там видно, что и когда было опубликовано (или почему пропущено/не удалось).

Изменения в панели подхватываются worker-процессом автоматически в течение
минуты (без перезапуска) — он периодически сверяется с базой.

## Локальная разработка без Docker

```
python -m venv .venv
.venv\Scripts\activate      # Windows
pip install -r requirements.txt
copy .env.example .env      # заполнить вручную; DATABASE_URL можно указать на sqlite+aiosqlite:///./dev.db для быстрого старта
alembic upgrade head
uvicorn app.main:app --reload   # панель
python -m app.worker            # в отдельном терминале — воркер
```

## Известные ограничения (первая версия)

Смотрите раздел «Явно вне scope» в [PROJECT_SPEC.md](PROJECT_SPEC.md). Отдельно стоит знать:

- Отложенные задания на публикацию комментариев хранятся в памяти worker-процесса (APScheduler без persistent jobstore) — при перезапуске worker уже запланированные, но ещё не опубликованные комментарии из текущей очереди теряются (новые посты по-прежнему подхватятся). Приемлемо для личного использования; при необходимости можно добавить `SQLAlchemyJobStore`.
- Резолвинг канала при добавлении и джойн канала используют временное подключение через выбранный аккаунт — сам аккаунт должен либо уже состоять в канале, либо канал должен быть публичным по username.
